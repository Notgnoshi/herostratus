This is a Rust project to generate achievements for Git repositories.

There is an `ARCHITECTURE.md` document that outlines the project architecture.

See `docs/developer/quickstart.md` for build, test, and run instructions.

The primary logic for processing a repository is in `herostratus/src/achievement/`. The definition
of a `Rule` to graint `Achievement`s, along with the implementation of the `Rule`s is in
`herostratus/src/rules/`.

A list of `Rule`s is maintained at `RULES.md` as a form of user documentation.

There are design documents in `docs/design/`.

There are integration tests in `herostratus/tests/`, which build and run the herostratus binary on
test fixtures generated by utilities in `herostratus-tests/`.

# Visibility conventions

Visibility specifiers (`pub(crate)`, `pub(super)`, `pub(in ...)`) should rarely, if ever, be used.
Seeing one should be a "PAY ATTENTION!" flag that something unusual is going on. Follow these rules:

* A symbol should only be `pub` if it is used outside its module. Otherwise, keep it private.
* If a type is shared outside a module tree, re-export it up to the top of the tree.
* Prefer `mod.rs` files that contain just module definitions and re-exports.
* Prefer private modules with public re-exports, unless a public module helps with scoping (e.g.,
  `tests::fixtures::repository` -- there are more fixtures than just `repository`, so use `pub mod`
  rather than re-exporting).
